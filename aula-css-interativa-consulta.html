Módulo 1 — O que é CSS e como ele funciona?

Básico
▼

O que é CSS?

CSS (Cascading Style Sheets) é a linguagem que diz como seu HTML vai aparecer na tela. Enquanto o HTML é o esqueleto, o CSS é a roupa, a maquiagem e a decoração.

As 3 formas de usar CSS

1. Inline — diretamente no elemento HTML (evite, exceto em casos pontuais):

<p style="color: red; font-size: 20px;">Texto vermelho</p>

2. Internal — dentro de uma tag <style> no <head>:

<style> p { color: red; } </style>

3. External — arquivo .css separado, linkado no HTML (o jeito certo!):

<link rel="stylesheet" href="estilo.css">

Anatomia de uma regra CSS

seletor { propriedade: valor; outra-propriedade: outro-valor; } /* Exemplo real: */ h1 { color: royalblue; font-size: 32px; text-align: center; }
💡
O C de "Cascading" significa cascata: quando dois estilos conflitam, o CSS tem regras para decidir qual vence (especificidade + ordem).

Módulo 2 — Seletores: como apontar para elementos

Básico
▼

Tipos de seletores

/* 1. Elemento — pega TODAS as tags desse tipo */ p { color: blue; } h1 { font-size: 2rem; } /* 2. Classe — pont para class="nome" no HTML */ .destaque { background: yellow; } .btn-primario{ background: blue; color: white; } /* 3. ID — aponta para id="nome" (único na página) */ #cabecalho { background: black; } /* 4. Universal — todos os elementos */ * { margin: 0; padding: 0; } /* 5. Combinadores */ div p { color: red; } /* todo <p> DENTRO de div */ div > p { color: red; } /* filho DIRETO */ h1 + p { color: red; } /* irmão imediato */ /* 6. Pseudo-classes — estado do elemento */ a:hover { color: orange; } /* mouse por cima */ input:focus { border: 2px solid blue; } li:first-child{ font-weight: bold; } /* 7. Pseudo-elementos — parte do elemento */ p::first-line{ font-weight: bold; } h2::before { content: "🔷 "; }

Especificidade

Quando dois seletores entram em conflito, quem tem maior especificidade vence:

/* Força crescente: */ * → 0 /* universal */ elemento → 1 /* ex: p, h1, div */ .classe → 10 /* ex: .btn */ #id → 100 /* ex: #header */ style="" → 1000/* inline */ !important → ∞ /* EVITE usar! */

Módulo 3 — Box Model: a caixa de todo elemento

Básico
▼

Todo elemento HTML é tratado como uma caixa retangular com 4 camadas:

margin (espaço externo)
border (borda)
padding (espaço interno)
conteúdo
.caixa { /* CONTEÚDO */ width: 200px; height: 100px; /* PADDING — espaço INTERNO (entre conteúdo e borda) */ padding: 20px; /* todos os lados */ padding: 10px 20px; /* cima/baixo | esq/dir */ /* BORDER — a borda visível */ border: 2px solid #333; /* espessura | estilo | cor */ border-radius: 8px; /* cantos arredondados */ /* MARGIN — espaço EXTERNO (empurra outros elementos) */ margin: 16px; margin: 0 auto; /* centraliza horizontalmente! */ /* BOX-SIZING (use sempre!) */ box-sizing: border-box; /* padding/border ficam DENTRO do width */ }
⚠️
Por padrão, width: 200px só conta o conteúdo. Com box-sizing: border-box, padding e border ficam dentro dos 200px — muito mais previsível!

Módulo 4 — Tipografia, Cores e Fundos

Básico
▼

Tipografia

p { font-family: 'Segoe UI', Arial, sans-serif; /* pilha de fontes */ font-size: 16px; /* ou rem (relativo ao root) */ font-weight: 400; /* 100–900 | normal | bold */ font-style: italic; line-height: 1.6; /* espaço entre linhas */ letter-spacing: 0.05em; /* espaço entre letras */ text-align: center; /* left | right | justify */ text-transform: uppercase; /* capitalize | lowercase */ text-decoration: underline; /* none | line-through */ } /* Google Fonts — adicione no <head>: */ /* <link href="https://fonts.googleapis.com/css2?family=Inter&display=swap" rel="stylesheet"> */ body { font-family: 'Inter', sans-serif; }

Cores

.exemplo { /* 4 formas de definir cor: */ color: red; /* nome */ color: #ff0000; /* hex */ color: rgb(255, 0, 0); /* rgb */ color: hsl(0, 100%, 50%); /* hsl — mais intuitivo! */ color: rgba(255,0,0, 0.5); /* com transparência */ /* Gradientes no background! */ background: linear-gradient(135deg, #667eea, #764ba2); background: radial-gradient(circle, #ff6b6b, #feca57); }

Fundos (background)

.card { background-color: #f0f0f0; background-image: url('imagem.jpg'); background-size: cover; /* cover | contain | 100px */ background-position: center; background-repeat: no-repeat; }

Módulo 5 — Flexbox: layouts em linha ou coluna

Intermediário
▼

O Flexbox é o jeito moderno de alinhar e distribuir elementos em uma dimensão (linha ou coluna).

Item 1
Item 2
Item 3
/* No PAI (container): */ .container { display: flex; flex-direction: row; /* row | column | row-reverse */ justify-content: center; /* eixo PRINCIPAL: flex-start | flex-end | space-between | space-around */ align-items: center; /* eixo CRUZADO: flex-start | flex-end | stretch */ flex-wrap: wrap; /* quebra linha se não couber */ gap: 16px; /* espaço entre os filhos */ } /* Nos FILHOS (itens): */ .item { flex: 1; /* cresce pra preencher o espaço */ order: 2; /* reordena sem mexer no HTML */ align-self: flex-end; /* sobrescreve align-items para este item */ }
💡
Para centralizar um elemento na tela: display: flex; justify-content: center; align-items: center; no pai — o truque mais usado do CSS!

Módulo 7 — Design Responsivo e Media Queries

Intermediário
▼

O que é design responsivo?

O site precisa funcionar bem em celular, tablet e desktop. A ferramenta principal são as Media Queries.

/* Abordagem "Mobile First" — comece pelo celular, expanda */ /* Estilos base (celular): */ .grid { display: grid; grid-template-columns: 1fr; /* 1 coluna no celular */ } /* Tablet (≥ 768px): */ @media (min-width: 768px) { .grid { grid-template-columns: repeat(2, 1fr); } } /* Desktop (≥ 1024px): */ @media (min-width: 1024px) { .grid { grid-template-columns: repeat(3, 1fr); } } /* Preferência por tema escuro do sistema: */ @media (prefers-color-scheme: dark) { body { background: #111; color: #eee; } }

Unidades relativas

/* px — absoluto, não escala */ /* % — relativo ao pai */ /* em — relativo ao font-size do elemento */ /* rem — relativo ao font-size do :root (mais previsível!) */ /* vw — % da largura da viewport */ /* vh — % da altura da viewport */ html { font-size: 16px; } /* base */ h1 { font-size: 2rem; } /* = 32px */ section { min-height: 100vh; } /* altura total da tela */

Módulo 8 — Transições e Animações

Intermediário
▼

Transition — animação ao mudar estado

.botao { background: royalblue; transform: scale(1); transition: all 0.3s ease; /* propriedade | duração | timing */ } .botao:hover { background: darkblue; transform: scale(1.05); /* cresce 5% */ box-shadow: 0 8px 20px rgba(0,0,0,.3); }

Transform — transformações 2D/3D

.elem { transform: translate(20px, 10px); /* move */ transform: rotate(45deg); /* gira */ transform: scale(1.5); /* escala */ transform: skew(10deg); /* inclina */ transform: translate(-50%, -50%); /* truque de centralização! */ }

@keyframes — animação livre

@keyframes pulsar { 0% { transform: scale(1); opacity: 1; } 50% { transform: scale(1.1); opacity: .7; } 100% { transform: scale(1); opacity: 1; } } .pulsando { animation: pulsar 2s ease-in-out infinite; /* nome | duração | timing | repetição */ } @keyframes entrar { from { opacity: 0; transform: translateY(30px); } to { opacity: 1; transform: translateY(0); } }

Módulo 9 — Variáveis CSS e Organização

Avançado
▼

Custom Properties (Variáveis CSS)

Defina valores reutilizáveis e mantenha o código consistente:

/* Declare no :root para serem globais */ :root { --cor-primaria: #7c3aed; --cor-texto: #1e293b; --fonte-base: 16px; --radius: 12px; --sombra: 0 4px 16px rgba(0,0,0,0.1); } /* Use com var() */ .botao { background: var(--cor-primaria); border-radius: var(--radius); box-shadow: var(--sombra); } /* Altere no JavaScript! */ document.documentElement.style .setProperty('--cor-primaria', '#e11d48');

BEM — Convenção de nomenclatura de classes

/* BEM = Bloco__Elemento--Modificador */ .card { } /* bloco */ .card__titulo { } /* elemento do card */ .card__imagem { } /* elemento do card */ .card--destaque { } /* modificador: card em destaque */ .card--desativado { } /* modificador: card inativo */

Pseudo-elementos úteis

.separador::before { content: ""; /* obrigatório para exibir */ display: block; width: 50px; height: 3px; background: var(--cor-primaria); margin: 0 auto 16px; }

Módulo 10 — Frameworks CSS: Tailwind, Bootstrap e mais

Avançado
▼

Por que usar um framework?

Frameworks oferecem componentes e utilitários prontos, acelerando o desenvolvimento. Mas só fazem sentido depois que você já entende CSS — o que você está fazendo agora!

🥇 Tailwind CSS — Utility First

Você aplica classes de utilitário diretamente no HTML. Muito popular atualmente.

<!-- Instalar: npm install -D tailwindcss --> <!-- Uso: classes utilitárias no HTML --> <button class="bg-blue-600 text-white px-6 py-3 rounded-lg hover:bg-blue-700 transition-all shadow-md"> Clique aqui </button> <!-- Grid responsivo --> <div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6"> ... </div>

🥈 Bootstrap — Component First

Componentes prontos (navbar, cards, modais...). Mais fácil de começar, menos flexível.

<!-- Via CDN no <head>: --> <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap@5/dist/css/bootstrap.min.css"> <!-- Uso --> <button class="btn btn-primary btn-lg">Botão</button> <div class="row"> <div class="col-md-4">...</div> <div class="col-md-8">...</div> </div>

Outros que vale conhecer

  • Bulma — leve, baseado em Flexbox, sem JavaScript
  • Sass/SCSS — CSS com variáveis, funções, aninhamento (compile para CSS)
  • CSS Modules — escopa estilos por componente (usado com React/Vue)
  • Styled Components — CSS dentro do JavaScript (React)
🗺️
Minha sugestão de caminho: CSS Puro → Tailwind CSS → (depois) SCSS para projetos grandes.
